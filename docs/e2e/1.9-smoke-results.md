# Task 1.9 — E2E Smoke Results (2026-05-29)

## Method
GUI simulator (evenhub-simulator Electron app) could NOT launch in this
environment — it exits immediately with macOS window-server/LaunchServices
XPC errors and never opens automation port 9898. Used the documented
fallback: dev-server serve check + headless WS round-trip using the app's
exact config (.env.local LAN URL + token) and exact frame shapes, then a
deterministic replay of the live frames through the app's reduce()+renderChat().

## (i) Dev server serves the app
http://localhost:5173 returns the built index.html; /src/main.ts -> HTTP 200.

## (ii) Live headless round-trip (ws://10.0.0.2:8765)
hello{token,device:g2} -> hello.ok -> tool.start(date) -> tool.end(terminal ok=true)
 -> assistant x8 (streamed) -> turn.done
ASSISTANT_REPLY: "It's Friday, May 29, 2026 — 5:31 PM CDT."

## Reducer/view replay (what the glasses would show)
INITIAL    body="Single-tap to ask"  status="connecting ws://10.0.0.2:8765"
CONNECTED  body="Single-tap to ask"  status="connected"
streaming  body grows to full reply
FINAL      body="It's Friday, May 29, 2026 — 5:31 PM CDT."  status="⚙ terminal ✓"   <-- BUG

## BUG FOUND
File: src/ui/views.ts (status expression) + src/state/store.ts (reducer).
Symptom: after a turn that used a tool, status is permanently stuck at the
tool indicator ("⚙ terminal ✓") and NEVER reaches "✓ done". The view picks the
tool branch whenever s.chat.tool is truthy; the reducer never clears chat.tool
(tool.end only sets running:false; turn.done only sets done:true). So the
done/conn branch is unreachable once any tool runs.
Body streaming itself works correctly.
